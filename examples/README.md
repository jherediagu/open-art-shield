# Example: a reproducible watermark audit

This directory contains a complete, reproducible OpenArtShield audit: an original
image, its watermarked version, and the JSON + HTML reports produced by the real
`oas` CLI.

```
examples/
  images/
    sample-original.png     # procedurally generated source image
    sample-protected.png    # same image with an invisible watermark embedded
  reports/
    sample-audit.json       # machine-readable audit report
    sample-audit.html       # standalone HTML version of the same report
  commands/
    generate-sample-image.mjs  # regenerates sample-original.png
    sample-audit.sh            # regenerates the protected image + reports
```

## The image

`sample-original.png` is **procedurally generated** by
[`commands/generate-sample-image.mjs`](commands/generate-sample-image.mjs) - a
512x512 image built from gradients, a soft glow, diagonal banding, a darker
"subject" rectangle, and film grain. It is deliberately textured because flat,
solid-color images are poor watermarking subjects (their mid-frequency DCT
coefficients are near zero).

It is safe to include here because it is **generated by our own script** (CC0 /
self-owned). No third-party or copyrighted artwork is used. That choice is
intentional: it would be contradictory to embed a "do not train on this" message
into an artist's work we don't own in order to demo the tool.

## How to reproduce

From the repository root, after `pnpm install`:

```bash
pnpm build

# (optional) regenerate the source image
node examples/commands/generate-sample-image.mjs

# embed the watermark + run the full audit, writing both reports
bash examples/commands/sample-audit.sh
```

The underlying commands are:

```bash
oas embed examples/images/sample-original.png \
  --message "artist=demo;license=no-ai-training" \
  --seed 123 \
  --strength 8 \
  --repetitions 5 \
  --out examples/images/sample-protected.png

oas audit examples/images/sample-original.png \
  --message "artist=demo;license=no-ai-training" \
  --seed 123 \
  --strength 8 \
  --repetitions 5 \
  --out examples/reports/sample-audit.json \
  --html examples/reports/sample-audit.html
```

Everything is deterministic (fixed seed), so re-running produces the same output.

## How to read this

- **Recovered** - did the CRC-32 checksum validate and the exact message come
  back? This is all-or-nothing.
- **Bit accuracy** - fraction of payload bits recovered correctly. Note that a
  high bit accuracy (e.g. 0.98) can still fail recovery: a few wrong bits break
  the checksum, so "almost" does not count.
- **PSNR / SSIM** - how visually close the transformed image is to the protected
  image. They measure image damage, not watermark survival, and are only
  reported when the transform preserved the dimensions.

A useful read of this particular run:

- The watermark is comfortable with **light JPEG, brightness, contrast, and mild
  blur**.
- It **breaks under heavier compression, downscaling, strong blur, cropping, and
  the screenshot-style pipeline**. `center_crop_90` is the worst case here
  (~0.5 bit accuracy) because cropping shifts the 8x8 block grid the watermark
  depends on.

## What the results do and do not mean

A "recovered" result means **only** this: the embedded signal survived that
specific transformation, at those specific parameters, for this specific image.

It does **not** mean:

- that the image is protected from AI training, copying, or transformation;
- that the watermark will survive other transforms, stronger attacks, or a
  motivated adversary running a removal tool;
- that recovery generalizes to other images, sizes, messages, or parameters.

OpenArtShield is a **measurement tool, not a defense**. Treat every result as
experimental. See the [root README](../README.md) and [ROADMAP](../ROADMAP.md)
for the project's framing and limitations.
